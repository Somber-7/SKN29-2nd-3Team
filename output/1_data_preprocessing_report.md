## 1. 프로젝트 개요

### 1-1. 비즈니스 배경

한국 부동산 시장은 2015~2023년 사이 급격한 가격 변동을 겪었습니다.  
국토교통부가 공개하는 아파트 실거래가 데이터는 500만 건 이상으로 방대하지만,  
일반 소비자나 투자자가 이를 직접 분석하기는 어렵습니다.

- 아파트 가격은 입지, 층, 면적, 브랜드, 금리 등 다양한 변수에 의해 결정됨
- 동일 지역 내에서도 브랜드·단지 규모에 따라 가격 격차가 존재
- 전국 246개 시군구에 걸친 거래 데이터를 직관적으로 이해하기 어려움
- 일부 거래는 시장 평균 패턴과 크게 다른 특이 거래로 분류될 수 있음

### 1-2. 비즈니스 목표

| 분석 목표 | 설명 |
|----------|------|
| 가격 예측 | 입지·단지 특성을 기반으로 아파트 거래금액 예측 |
| 브랜드 등급 분류 | 가격·입지 특성만으로 프리미엄/일반/공공/기타 등급 분류 |
| 지역 군집화 | 전국 거래 데이터를 입지·가격 특성 기준으로 권역 군집화 |
| 이상 거래 탐지 | 전국·서울 구별·시군구별 기준으로 특이 거래 탐지 |
| 프리미엄 분석 | 지역별 가격 프리미엄 패턴 분석 |

### 1-3. 머신러닝 활용 계획

| 항목 | 내용 |
|------|------|
| 회귀 (가격 예측) | Linear Regression, Random Forest, LightGBM, XGBoost, DNN(PyTorch) |
| 분류 (브랜드 등급) | XGBoost 다중 분류 (4개 등급) |
| 군집화 (지역 분류) | TorchKMeans GPU (k=7) |
| 이상 탐지 | IsolationForest (전국 단일 / 서울 구별 / 전국 시군구별) |
| 활용 방안 | Streamlit 대화형 웹 앱으로 분석 결과 시각화 제공 |
| 기대 효과 | 비전문가도 500만 건 데이터를 직관적으로 탐색·해석 가능 |

### 1-4. 성공 기준 (Success Criteria)

| 모델 | 주요 평가 지표 | 목표 |
|------|-------------|------|
| 회귀 (DNN) | R² | ≥ 0.95 |
| 회귀 (머신러닝) | R² | ≥ 0.90 |
| 분류 (브랜드 등급) | F1 (weighted) | ≥ 0.90 |
| 군집화 | Silhouette | ≥ 0.40 |
| 이상 탐지 | contamination 기준 탐지 일관성 | 3% 수준 유지 |

## 4. 데이터 전처리

### 4-1. 결측값 처리

| 컬럼명 | 처리 방법 | 근거 |
|--------|----------|------|
| 건축년도 | 중앙값(2004) 대체 | 이상치 영향 최소화, 결측 76건으로 소량 |
| 위도/경도 | 해당 행 제외 (군집화·이상탐지 시) | 좌표 없이 위치 기반 군집화 불가 |
| 세대수 | 중앙값(126,696) 대체 | 결측률 0.37%, 전국 중간 시군구 세대수로 대체 |
| 시군구 (260건) | 군위군 표기 적용 | 2023년 대구 편입 특수 케이스 — 별도 매핑 추가 |

### 4-2. 이상값 처리

| 컬럼명 | 처리 방법 | 근거 |
|--------|----------|------|
| 거래금액 | 유지 (문자열 쉼표 제거 후 float 변환) | 실제 신고된 거래가 — 이상치도 분석 대상 |
| 층 (음수) | 1로 대체 또는 절댓값 | 지하층 거래는 희소, 모델 혼란 방지 |

### 4-3. 인코딩 (범주형 → 수치형)

| 특성명 | 인코딩 방법 | 변환 결과 | 선택 근거 |
|--------|------------|----------|----------|
| 시군구 | One-Hot Encoding | 최대 248개 더미 변수 | 이상탐지 모델 지역 정보 반영 |
| 지역코드 | One-Hot Encoding | 전국 이상탐지 모델 | 상동 |
| 브랜드여부 | 원본 사용 (0/1) | — | 이미 이진 인코딩 |
| 거래금액 | str → float (쉼표 제거) | `85,000` → `85000.0` | 원본이 문자열 형태로 저장됨 |

### 4-4. 파생변수 생성 (Feature Engineering)

| 파생변수 | 계산식 | 역할 | 사용 모델 |
|---------|--------|------|---------|
| 평당가 | `거래금액 / (전용면적 / 3.3)` | 단위 면적당 가격 — 지역 프리미엄 반영 | 이상탐지, 군집화, 프리미엄 분석 |
| 건물연식 | `2026 - 건축년도` | 노후도 지표 | 회귀, 분류, 군집화, 이상탐지 |
| 거래연도 | `거래일.year` | 연도별 시장 트렌드 반영 | 회귀, 분류, 이상탐지 |
| 거래월 | `거래일.month` | 계절성 반영 | 회귀, 분류, 이상탐지 |
| 거래활성도 | `아파트별 거래건수 / 전체 거래건수` | 단지 인기도·유동성 | 군집화 |

### 4-5. 데이터 불균형 처리

분류 모델(브랜드 등급) 클래스 분포:

| 등급 | 비율 | 불균형 여부 |
|------|------|-----------|
| 기타 | 약 44% | — |
| 프리미엄 | 약 24% | — |
| 일반브랜드 | 약 24% | — |
| 공공(LH) | 약 8% | 소수 클래스 |

- 오버샘플링(SMOTE): 미적용
- **class_weight 조정**: XGBoost `scale_pos_weight` 대신 전체 500만 건 학습으로 자연스러운 분포 반영
- 결과: 공공(LH) F1 = 0.997 — 소수 클래스도 충분히 학습됨

### 4-6. 스케일링

| 방법 | 적용 모델 | 적용 특성 | 선택 근거 |
|------|----------|----------|----------|
| StandardScaler | DNN (PyTorch) | 모든 수치형 특성 | 신경망은 입력 스케일 민감 |
| 미적용 | XGBoost, LightGBM, RandomForest | — | 트리 기반 모델은 스케일 불변 |
| 자체 정규화 | TorchKMeans | 위도·경도에 ×5 가중치 | 지리 좌표 거리 척도 통일 |
