## 1. 모델링 전략

### 1-1. 평가 지표 선정

| 모델 유형 | 최우선 지표 | 보조 지표 | 선정 이유 |
|----------|-----------|----------|----------|
| 회귀 (가격 예측) | RMSE | MAE, R² | 금액 단위(만원)로 오차 해석 용이, 이상치 패널티 강함 |
| 분류 (브랜드 등급) | F1 (weighted) | Accuracy, Precision, Recall | 4개 클래스 불균형 존재 → 클래스별 균형 측정 |
| 군집화 (지역 군집) | Silhouette | Davies-Bouldin, Calinski-Harabasz | 군집 내 응집도 + 군집 간 분리도 종합 |
| 이상 탐지 | anomaly_score 분포 | 특이 거래 비율 | 정답 레이블 없음 → 점수 분포와 도메인 해석으로 판단 |

### 1-2. 후보 모델 선정

| 모델 | 유형 | 선정 근거 |
|------|------|----------|
| Linear Regression | 회귀 (선형) | 베이스라인 — 단순 선형 관계 기준선 |
| Random Forest | 회귀 (앙상블) | 비선형 관계 포착, 특성 중요도 제공 |

| DNN (PyTorch) | 회귀 (딥러닝) | 복잡한 비선형 특성 관계 포착 |
| TorchKMeans | 군집화 (딥러닝 GPU) | 500만 건 전체 배치 완전 수렴 보장 |
| IsolationForest | 이상 탐지 | 비지도 이상치 탐지 표준 알고리즘, 레이블 불필요 |

### 1-3. 실험 계획

1. 각 회귀 모델 기본 하이퍼파라미터로 베이스라인 성능 측정
2. XGBoost 분류기로 브랜드 등급 다중 분류 → Early Stopping 튜닝
3. TorchKMeans: MiniBatchKMeans 베이스라인 → GPU 전환 → k·가중치 탐색 (29개 실험)
4. IsolationForest: 전국/서울 구별/시군구별 3가지 관점 구성
5. 최종 선정 기준: 회귀 R² ≥ 0.95, 분류 F1 ≥ 0.90, Silhouette ≥ 0.40

비슷한 입지 특성을 가진 거래를 하나의 권역으로 묶습니다.

**최종 피처 구성**

| 피처 | 계산식 | 가중치 |

### 4-2. 알고리즘 선택: MiniBatchKMeans → TorchKMeans GPU

| 선택 이유 | — | 성능·속도 모두 우수 |

> 전체 데이터(490만×5피처 ≈ 100MB)를 GPU 메모리에 올려 완전 수렴 보장.

### 6-1. PricePremiumAnalyzer 목적 및 방법론

- **목적**: 특정 지역·브랜드·시기의 거래가격이 주변 평균 대비 얼마나 높은지(프리미엄) 분석
- **방법**: 지역별 평당가 평균 산출 → 개별 거래와의 차이 계산 → 프리미엄 지수 생성
- **결과 저장**: `data/models/premium_analysis_results.pkl` (214KB)
- **Streamlit 활용**: 지역별 프리미엄 히트맵, 브랜드별 프리미엄 비교 차트

### 6-2. 분석 결과 요약
