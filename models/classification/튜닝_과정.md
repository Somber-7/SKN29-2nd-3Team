## 단계별 튜닝 내역

### 1단계 — 메모리 오류 수정
**문제**
- `OneHotEncoder`로 `지역코드`(정수) + `시군구`(~250개)를 원핫 인코딩 시 14.9 GiB dense 배열 생성으로 OOM 발생

**해결**
- `지역코드` → `numeric_cols`로 이동 (정수형이므로 수치 스케일링만)
- `시군구` → `OrdinalEncoder` 교체 (컬럼 1개로 인코딩)

---

### 2단계 — 클래스 불균형 해소
**문제**
- `기타` 32,592건 vs `일반브랜드` 7,127건 (약 4.6:1 불균형)
- `일반브랜드` Recall 0.254, `프리미엄` Recall 0.569

**시도 ①: 완전 역비율 가중치**
```python
class_weight = len(y_train) / (len(class_counts) * class_counts)
```
- `일반브랜드` Recall 0.254 → 0.702 (개선)
- `프리미엄` Precision 0.799 → 0.440 (과잉 보정 — 소수 클래스 과예측)

**시도 ②: sqrt 완화 가중치 (채택)**
```python
class_weight = np.sqrt(len(y_train) / (len(class_counts) * class_counts))
```
- Accuracy 0.800, F1 0.798 — 가중치 없음과 거의 동등
- 소수 클래스 Recall 개선 + Precision 균형 유지

---

### 3단계 — 키워드 목록 보강
**문제**
- `일반브랜드` Recall 0.461 — 키워드에 없는 건설사가 `기타`로 잘못 분류

**조치**
- DB `brand` 컬럼 unique 값 전수 조사
- 누락 브랜드 발견:
  - 프리미엄 키워드에 건설사명(`현대건설`, `삼성물산` 등) 추가 → 오히려 성능 하락
  - 건설사명은 프리미엄 브랜드 외 일반 아파트도 시공하므로 **일반브랜드로 이동**
- `일반브랜드`에 추가: `대림산업`, `삼익주택`, `경남기업`, `동원개발`, `신동아건설`, `동부건설`, `삼부토건`, `현진건설`, `금강주택`, `성지건설`, `광양건설`, `라온건설`, `아남건설`, `현대건설`, `삼성물산`, `포스코이앤씨`, `GS건설`, `HDC현대산업개발`, `DL이앤씨`, `롯데건설`

**결과**
- `기타` support 32,592 → 27,701 (약 5,000건 올바른 클래스로 이동)
- `일반브랜드` F1 0.556 → 0.632

---

### 4단계 — 하이퍼파라미터 조정
**변경**
- `n_estimators`: 300 → 500
- `max_depth`: 6 → 8

**결과**
- Accuracy 0.745 → **0.908**
- 일반브랜드 F1 0.632 → **0.855**
- 모든 클래스 F1 0.82 이상

---

### 5단계 — Early Stopping 적용 (최종)
**변경**
```python
n_estimators=1000,
early_stopping_rounds=30,
```
- train의 10%를 validation set으로 분리
- validation loss 기준 30라운드 개선 없으면 자동 중단

**결과**
- Accuracy **0.960**, F1 **0.960**
- `best_iteration=999` — 1000 상한에 도달 (early stopping 미발동)
- 트리 수가 많을수록 성능이 계속 향상되는 상태

---

## 최종 모델 설정

```python
XGBClassifier(
    objective="multi:softprob",
    n_estimators=1000,
    learning_rate=0.05,
    max_depth=8,
    subsample=0.8,
    colsample_bytree=0.8,
    early_stopping_rounds=30,
    eval_metric="mlogloss",
    device="cuda",  # GPU 자동 감지
)
```

- 클래스 가중치: `sqrt(n / (k * count))` 역비율 완화
- 인코딩: 수치형 StandardScaler + 범주형 OrdinalEncoder
