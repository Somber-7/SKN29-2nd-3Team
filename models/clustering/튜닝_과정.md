## 단계별 튜닝 과정

### 0단계 — 알고리즘 교체 (PyTorch KMeans → sklearn MiniBatchKMeans)

- PyTorch 전체 배치 KMeans: K-Means++ 초기화 시 490만 건 `torch.cdist` 연산으로 수십 분 이상 소요 → 중단
- sklearn MiniBatchKMeans로 교체: 미니배치 방식으로 메모리 효율적, 약 50초 이내 학습 완료
- 초기 Silhouette 0.177

### 1단계 — 위도/경도 가중치 적용

- **변경**: StandardScaler 정규화 후 위도/경도에 ×3 가중치 적용
- **결과**: Silhouette 0.177 → 0.258 (+0.081)
- **이유**: 가중치 적용 전에는 평당가/건물연식 등 다른 피처들이 지리 정보를 희석

### 2단계 — 피처 축소 (7개 → 4개)

- **변경**: 인근학교수, 인근역수, 세대수 제거 → 위도, 경도, 평당가, 건물연식만 사용
- **결과**: Silhouette 0.258 → 0.262, DB 1.408 → 1.313
- **이유**: 위도/경도와 상관관계가 높은 피처들이 중복 정보로 작용 — 제거 시 노이즈 감소

### 3단계 — 추가 피처 조합 탐색

**시도 1: 인근학교수 + 인근역수 추가 (6개)**
- Silhouette 0.262 → 0.221 — 악화 (위도/경도와 중복 정보)

**시도 2: 전용면적 + 층 추가 (6개)**
- Silhouette 0.262 → 0.213 — 악화 (전용면적은 평당가에 이미 포함, 층은 단지 내 변동성 과대)

### 4단계 — 파생변수 설계

| 파생변수 | 계산식 | Silhouette (단독) |
|---|---|---|
| log_세대수 | log1p(세대수) | 0.297 |
| 접근성지수 | 인근역수 × 인근학교수 | 0.310 |
| **거래활성도** | **아파트별 거래건수 / 전체 거래건수** | **0.362** |

- 거래활성도 단독 추가가 압도적으로 효과적
- 둘 이상 조합 시 오히려 악화 — 유사한 지역 특성을 중복 표현
- predict 시 누수 방지: fit 시 `apt_activity_map_` 저장, predict 시 저장 맵으로 매핑

### 5단계 — MiniBatchKMeans 파라미터 튜닝

- batch_size 10,000 → 50,000: 모든 지표 악화
- n_init 3 → 10: Silhouette/CH 하락, DB만 소폭 개선
- **MiniBatchKMeans 최종: Silhouette 0.362** (미니배치 특성상 수렴이 불완전)

### 6단계 — 알고리즘 변경 탐색 (PCA, 연도별)

- PCA 3차원: Silhouette 0.362 → 0.322 하락 — 효과 없음
- 연도별 군집화: 평균 Silhouette 0.310 — 전체 학습보다 낮음

### 7단계 — PyTorch GPU 전체 배치 KMeans 재도입

- K-Means++ 대신 **랜덤 초기화** 사용 (hanging 문제 해결)
- 거리 계산을 전체 한 번에 처리 (OOM 시 배치 폴백)
- 500만×5피처×float32 ≈ 100MB, 거리 행렬 ≈ 140MB — RTX 5060 8GB에서 여유
- MiniBatch 대비 완전한 수렴 보장

**가중치 탐색 (n_init=10 고정):**
- ×3: Silhouette 0.3277 (MiniBatch보다 낮음 — 수렴 차이로 이후 개선 여지 확인)
- ×5: Silhouette 0.4694 ← **최적 균형점**
- ×7~×15: 지표는 계속 상승하나 위도/경도만으로 군집화하는 것과 동일해짐
  - ×10~×15에서 일부 군집이 완전히 고정(거래건수 변화 수백 건 이하) — 포화 상태

**n_init 탐색:**
- ×3 가중치 공간에서 n_init 5→10: 완전히 동일한 결과 — local minimum이 하나
- ×5 가중치 공간에서 n_init 5→10: Silhouette 0.4496 → 0.4694 추가 개선
  - 가중치가 피처 공간을 바꾸어 multiple local minima 생성 → n_init 효과 발생

**k 재탐색 (위도/경도 ×5, n_init=10 고정):**
- k=5: Silhouette 0.5100 (최고) — 수도권 47% 단일 군집, 5개 권역으로 분리
- k=7: Silhouette 0.4694 — 부산·경남 2개 세분화, 전남 동부(순천·여수·광양) 별도 분리
- k=9 이상: 급격히 악화, 학습 시간 폭증 (수렴 불안정)
- **k=7 선택 이유**: k=5 대비 Silhouette 0.04 낮지만 해석 가능성이 훨씬 풍부

**거래활성도 가중치 추가 탐색 (k=7, 위도/경도 ×5):**
- ×1.5~×5.0 전 구간에서 기준(0.4694)을 넘지 못함 — 효과 없음

## 핵심 인사이트

1. **GPU 전체 배치가 MiniBatch보다 수렴 품질이 높음** — 미니배치는 구조적으로 불완전 수렴, GPU 전체 배치로 0.362 → 0.4694 달성
2. **가중치는 피처 공간 자체를 바꿈** — ×5에서 임계점을 넘어 n_init 효과가 비로소 발생
3. **가중치 상한이 존재** — ×7 이상에서 지표는 상승하나 위도/경도만으로 군집화하는 것과 동일해져 다른 피처 정보가 소실
4. **거래활성도는 피처로는 유효하나 가중치 추가는 무의미** — 이미 피처 공간에서 충분히 기여
5. **k=5 vs k=7 트레이드오프** — 지표는 k=5(0.5100)가 높지만 해석 가능성은 k=7이 압도적

## Silhouette 0.4694 평가

일반적인 Silhouette 기준:
- 0.7 이상: 강한 군집 구조
- 0.5~0.7: 적당한 군집 구조
- 0.3~0.5: 약한 군집 구조

0.4694는 "약한~적당한" 경계 수준이나, **전국 500만 건 연속 분포 데이터 특성상 자연적 군집 경계가 불분명**하여 이 수준이 현실적 최선입니다. 초기 0.177 대비 **2.65배 개선**했으며, 군집별 지역 해석력(서울·수도권 / 부산 외곽 / 충청·전북 / 창원·김해 / 전남·광주 / 강원·경북 / 전남 동부)은 충분히 확보됨.
