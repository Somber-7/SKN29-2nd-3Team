# 데이터베이스 구성 정리

## 파일 목록

```
SKN29-2nd-3Team/
├── conf/
│   └── .env                     ← DB 접속 정보
├── sql/
│   └── ddl/
│       └── create_db.sql        ← DB/테이블 생성 DDL
├── scripts/
│   └── insert_data.py           ← CSV → MySQL 배치 INSERT 스크립트

```

## 접속 정보 (conf/.env)

```env
DB_HOST     = localhost
DB_USER     = root
DB_PASSWORD = root1234!
DB_NAME     = apart_deal
DB_PORT     = 3306
```

---

## 초기 설정 순서

```bash
# 1. DB 및 테이블 생성
mysql -u root -p < sql/ddl/create_db.sql

# 2. CSV 데이터 INSERT (약 10~20분 소요)
python scripts/insert_data.py
```

## tbl_apart_deals 컬럼 정의

### 지역 정보

| 컬럼명 | 타입 | NULL | 원본 컬럼 | 설명 |
|--------|------|------|----------|------|
| `id` | BIGINT | NOT NULL | — | AUTO_INCREMENT PK |
| `region_code` | INT | NOT NULL | 지역코드 | 행정구역 코드 (5자리) |
| `sigungu` | VARCHAR(50) | NULL | 시군구 | 시/군/구 명칭 |
| `dong` | VARCHAR(50) | NOT NULL | 법정동 | 법정동 명칭 |
| `jibun` | VARCHAR(20) | NULL | 지번 | 부번 포함 (예: 95-15) |

### 단지 정보

| 컬럼명 | 타입 | NULL | 원본 컬럼 | 설명 |
|--------|------|------|----------|------|
| `apt_name` | VARCHAR(100) | NULL | 아파트 | 단지명 |
| `brand` | VARCHAR(50) | NOT NULL | 브랜드 | 브랜드명, 기본값 `'기타'` |
| `is_brand` | TINYINT | NOT NULL | 브랜드여부 | 브랜드 유무 0/1, 기본값 `0` |
| `build_year` | SMALLINT | NULL | 건축년도 | 준공 연도 |
| `household_cnt` | INT | NULL | 세대수 | 단지 전체 세대수 |

### 거래 정보

| 컬럼명 | 타입 | NULL | 원본 컬럼 | 설명 |
|--------|------|------|----------|------|
| `deal_date` | DATE | NOT NULL | 거래일 | YYYY-MM-DD |
| `floor` | SMALLINT | NULL | 층 | 음수 = 지하층 (예: -1 = B1) |
| `area` | DECIMAL(8,2) | NULL | 전용면적 | 전용면적 (㎡) |
| `deal_amount` | INT | NULL | 거래금액 | 만원 단위, 쉼표 제거 후 저장 |

### 거시 지표 / 위치 / 입지

| 컬럼명 | 타입 | NULL | 원본 컬럼 | 설명 |
|--------|------|------|----------|------|
| `base_rate` | DECIMAL(5,2) | NULL | 기준금리 | 거래 시점 기준금리 (%) |
| `latitude` | DECIMAL(12,9) | NULL | 위도 | WGS84 위도 |
| `longitude` | DECIMAL(12,9) | NULL | 경도 | WGS84 경도 |
| `school_cnt` | SMALLINT | NULL | 인근학교수 | 반경 내 학교 수 |
| `station_cnt` | SMALLINT | NULL | 인근역수 | 반경 내 지하철역 수 |

### 인덱스

| 인덱스명 | 컬럼 |
|---------|------|
| PRIMARY | `id` |
| `idx_region_code` | `region_code` |
| `idx_sigungu` | `sigungu` |
| `idx_apt_name` | `apt_name` |
| `idx_deal_date` | `deal_date` |
| `idx_brand` | `brand` |
| `idx_deal_amount` | `deal_amount` |

## utils/db.py 사용 예시

```python
from utils.db import fetch_all, fetch_one

# 특정 지역 거래 조회
rows = fetch_all(
    "SELECT apt_name, deal_amount, deal_date FROM tbl_apart_deals WHERE sigungu = %s LIMIT 100",
    ("강남구",)
)

# 단지별 평균 거래금액
rows = fetch_all("""
    SELECT apt_name, AVG(deal_amount) AS avg_price, COUNT(*) AS cnt
    FROM tbl_apart_deals
    WHERE sigungu = %s
    GROUP BY apt_name
    ORDER BY avg_price DESC
""", ("강남구",))
```
