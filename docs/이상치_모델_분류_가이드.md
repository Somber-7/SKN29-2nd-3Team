Apart Deal_6.csv 전체 데이터 기준 이상 거래 탐지 모델 3개의 분류 방식 및 분석 흐름 정리입니다.

## 모델 3개 비교 개요

| 항목 | `anomaly_transaction_model` | `seoul_anomaly_transaction_model` | `location_anomaly_transaction_model` |
|------|-----------------------------|------------------------------------|---------------------------------------|
| 클래스명 | `AnomalyTransactionModel` | `SeoulDistrictAnomalyModel` | `LocationAnomalyModel` |
| 탐지 범위 | 전국 전체 | 서울 25개 구 | 전국 246개 시군구 |

## 공통 전처리 흐름

3개 모델 모두 동일한 전처리 파이프라인(`prepare_dataframe`)을 거칩니다.

### 처리 순서

```
1. CSV 로드
   └─ encoding="utf-8", low_memory=False

2. 거래금액 쉼표 제거
   └─ df["거래금액"] = df["거래금액"].astype(str).str.replace(",", "").astype(float)

3. 중복 행 제거
   └─ df.drop_duplicates(keep="first").reset_index(drop=True)

4. 필수 컬럼 존재 여부 검증
   └─ 13개 컬럼 누락 시 ValueError 발생

5. 숫자형 컬럼 안전 변환
   └─ 쉼표 포함 문자열 → float (변환 불가 값은 NaN)

6. 파생 피처 생성 (4개)
   ├─ 평당가   = 거래금액 / (전용면적 / 3.3)
   ├─ 건물연식 = 2026 - 건축년도
   ├─ 거래연도 = 거래일에서 연도 추출
   └─ 거래월   = 거래일에서 월 추출

7. 원본 컬럼 제거
   └─ 건축년도, 거래일 컬럼 drop (파생 피처로 대체)

8. 전용면적 0 이하 → NaN 처리 (평당가 계산 오류 방지)

9. inf / -inf → NaN 치환 후 피처 컬럼 기준 NaN 행 제거
```

### 파생 피처 계산식

| 파생 피처 | 계산식 | 비고 |
|-----------|--------|------|
| `평당가` | `거래금액 / (전용면적 / 3.3)` | 전용면적 ≤ 0이면 NaN |
| `건물연식` | `2026 - 건축년도` | current_year 파라미터로 조정 가능 |
| `거래연도` | `pd.to_datetime(거래일).dt.year` | 변환 불가 시 NaN |
| `거래월` | `pd.to_datetime(거래일).dt.month` | 변환 불가 시 NaN |

## 모델별 분류 방식

### 1. AnomalyTransactionModel — 전국 단일 모델

**분류 기준:** 전국 평균 거래 패턴 대비 절대 이상치

#### 피처 구성 (13종)

| 구분 | 피처 |
|------|------|
| 수치형 (13개) | 거래금액, 전용면적, 평당가, 층, 건물연식, 기준금리, 위도, 경도, 인근학교수, 인근역수, 세대수, 거래연도, 거래월 |

지역코드·시군구는 피처로 사용하지 않으며, 하나의 단일 기준선으로 전체를 판단합니다.

#### 학습 흐름

```
전체 데이터
  └─ 무작위 샘플링 (최대 20만 건, random_state=42)
       └─ StandardScaler (수치형 13개)
            └─ IsolationForest (n_estimators=200, contamination=0.03)
                 └─ 단일 모델 1개 완성
```

#### 탐지(추론) 흐름

```
전체 데이터를 10만 건씩 배치 처리
  └─ 각 배치에 학습된 단일 모델 적용
       └─ 결과 병합 후 anomaly_rank 전체 기준 재계산
```

### 2. SeoulDistrictAnomalyModel — 서울 구별 모델

**분류 기준:** 서울 25개 구 각각의 내부 패턴 대비 상대 이상치

#### 서울 데이터 필터 조건

```python
code_mask = df["지역코드"].str.startswith("11")   # 지역코드 앞 2자리가 11
name_mask = df["시군구"].str.contains("서울")      # 시군구에 "서울" 포함
# 둘 중 하나라도 해당하면 서울 데이터로 분류
```

#### 구 이름 추출

```python
# "서울 강남구" → "강남구"
구명 = 시군구.str.replace(r"^서울\s*", "", regex=True)
```

#### 피처 구성 (13종)

| 구분 | 피처 |
|------|------|
| 수치형만 (13개) | 거래금액, 전용면적, 평당가, 층, 건물연식, 기준금리, 위도, 경도, 인근학교수, 인근역수, 세대수, 거래연도, 거래월 |
| 범주형 | **제외** (구별 모델 내에서 지역코드·시군구는 상수) |

#### 학습 흐름

```
서울 필터링된 데이터
  └─ 구명 기준 groupby (최대 25개 구)
       └─ 구별 샘플 수 < 30 → 학습 제외 (skipped_districts_에 기록)
            └─ 구별 StandardScaler → IsolationForest (n_estimators=100)
                 └─ 구별 모델 딕셔너리 저장 {"강남구": Pipeline, ...}
```

#### 탐지(추론) 흐름

```
서울 필터링된 데이터
  └─ 구명 기준 groupby
       ├─ 학습된 구 → 해당 구 모델 적용 → anomaly_score / anomaly_rank (구 내 순위)
       └─ 학습 안 된 구 → anomaly_label = "미탐지(학습데이터부족)"
            └─ 전체 결합 후 district_rank (서울 전체 기준 순위) 추가 계산
```

### 3. LocationAnomalyModel — 전국 시군구별 모델

**분류 기준:** 전국 246개 시군구 각각의 내부 패턴 대비 상대 이상치

SeoulDistrictAnomalyModel의 전국 확장판으로, 서울 외 지방 소도시까지 커버합니다.

#### 피처 구성 (13종)

| 구분 | 피처 |
|------|------|
| 수치형만 (13개) | 거래금액, 전용면적, 평당가, 층, 건물연식, 기준금리, 위도, 경도, 인근학교수, 인근역수, 세대수, 거래연도, 거래월 |
| 범주형 | **제외** (시군구별 모델 내에서 지역코드는 상수 → drop) |

#### 학습 흐름

```
전체 데이터 (지역코드 컬럼 제거)
  └─ 시군구 기준 groupby (최대 246개)
       └─ 시군구별 샘플 수 < 30 → 학습 제외
          (실행 결과: 경상북도 울릉군, 전라남도 신안군 제외 → 244개 학습)
            └─ 시군구별 StandardScaler → IsolationForest (n_estimators=100)
                 └─ 시군구별 모델 딕셔너리 저장 {"강남구": Pipeline, ...}
```

#### 탐지(추론) 흐름 및 TOP N 도출 방식

```
전체 데이터
  └─ 시군구 기준 groupby
       ├─ 학습된 시군구 → 해당 시군구 모델 적용 → anomaly_rank (시군구 내 순위)
       └─ 학습 안 된 시군구 → anomaly_label = "미탐지(학습데이터부족)"
            └─ 전체 결합 후 total_rank (전국 기준 순위) 추가 계산

TOP N 도출 (top_anomalies_top1_per_region)
  Step 1. 246개 시군구별로 anomaly_score 최솟값 1건씩 추출 (최대 246행)
  Step 2. 추출된 246건 중 anomaly_score 오름차순 상위 N건 반환
```

## anomaly_score 해석 기준

> 모델마다 score 범위가 다르므로 서로 다른 모델의 score를 직접 비교하면 안 됩니다.

| 범위 | 의미 |
|------|------|
| 음수, 절댓값 클수록 | 더 특이한 거래. 이상치에 가까움 |
| -0.15 이하 | 강한 이상 신호 (`location` 모델 기준 TOP권) |
| -0.10 ~ -0.15 | 중간 이상 신호 (`seoul` 모델 기준 TOP권) |
| -0.03 이하 | 약한 이상 신호 (`anomaly` 전국 단일 모델 기준 TOP권) |
| 0 근처 | 경계선 거래 |
| 양수 | 정상 패턴 거래 |
| `anomaly_raw_label = -1` | IsolationForest가 이상치로 분류 |
| `anomaly_rank = 1` | 해당 범위(전국/구/시군구) 내 가장 특이 |

## 주요 특이사항

### 1. 모델 결과는 통계적 이상치

이 모델이 탐지하는 결과는 **실제 불법·허위 거래가 아닙니다**.
발표 및 서비스에서는 반드시 **"모델 기준 특이 거래"**, **"일반 패턴과 다른 거래"** 로 표현하세요.

### 2. 모델간 score 직접 비교 불가

| 모델 | score 범위(TOP권 기준) |
|------|-----------------------|
| `anomaly` (전국 단일) | -0.02 ~ -0.03 |
| `seoul` (서울 구별) | -0.08 ~ -0.16 |
| `location` (전국 시군구별) | -0.13 ~ -0.16 |

세 모델은 학습 기준과 데이터 범위가 달라 동일 거래도 score 절댓값이 다르게 나옵니다.

### 3. min_samples 30 미만 시군구 학습 제외

거래 건수가 30건 미만인 시군구는 IsolationForest 학습에서 제외됩니다.
해당 시군구의 탐지 결과는 `anomaly_label = "미탐지(학습데이터부족)"` 로 표시됩니다.

실행 결과 기준: **경상북도 울릉군, 전라남도 신안군** 제외 → 244개 시군구 학습

### 4. 이상치는 고가뿐 아니라 저가도 탐지

해당 지역 평균과 크게 다른 방향이면 저가도 이상치로 분류됩니다.

```
고가 이상치 예시: 아크로서울포레스트 130억 (성동구 내 극단 고가)
저가 이상치 예시: 강북구 두온리치벨리 6억, 평당가 1,380 (서울 평균 대비 극단 저가)
```
