# 이상치 모델 작업 요약

아파트 실거래 데이터에서 일반적인 거래 패턴과 다른 특이 거래를 탐지하는 모델 3개 정리입니다.  
모든 모델은 `IsolationForest` 알고리즘을 사용하며, 탐지 범위와 비교 기준이 다릅니다.

## 분석 목적 및 위치

이 모델은 **예측 영역이 아닌 분석 영역**에 해당합니다.

- 예측 모델(회귀): 거래금액이 얼마일지 수치를 맞히는 것
- 이상치 분석: **유독 비싼 거래가 어디에 집중되는지**, **어떤 브랜드·단지가 반복 등장하는지** 패턴을 찾는 것

결과를 통해 다음 질문에 답할 수 있습니다.

```text
- 전국에서 가격 이상치가 집중된 지역은 어디인가?
- 시군구 내 평균 대비 가장 튀는 거래는 어떤 단지인가?
- 이상 거래에 반복 등장하는 브랜드는 무엇인가?
- 고가 이상 거래와 저가 이상 거래의 분포는 어떻게 다른가?
```

> 주의: 모델 결과는 실제 불법·허위 거래를 의미하지 않습니다.  
> 발표 및 서비스에서는 반드시 **"모델 기준 특이 거래"**, **"일반 패턴과 다른 거래"** 로 표현하세요.

## 2. anomaly_transaction_model

### 파일 위치

```text
models/anomaly/anomaly_transaction_model.py
```

### 클래스명

```python
AnomalyTransactionModel
```

### 목적

전국 아파트 실거래 데이터를 단일 모델로 학습하여 **전국 평균 패턴 대비 특이 거래**를 탐지합니다.  
지역코드와 시군구를 OneHot 인코딩으로 포함하므로 지역 정보가 반영되지만, 하나의 기준선으로 전체를 판단합니다.

### 주요 입력 피처

```text
거래금액      전용면적      평당가        층
건물연식      기준금리      위도          경도
인근학교수    인근역수      세대수        거래연도
거래월
(지역코드·시군구는 피처로 사용하지 않음)
```

### 내부 생성 피처

```text
평당가   = 거래금액 / (전용면적 / 3.3)
건물연식 = 2026 - 건축년도
거래연도 = 거래일에서 추출
거래월   = 거래일에서 추출
```

### 주요 결과 컬럼

```text
anomaly_raw_label   1=정상, -1=특이
anomaly_label       "정상거래" / "특이거래"
anomaly_score       값이 작을수록 더 특이 (전국 기준)
anomaly_rank        전국 기준 특이 순위 (1 = 가장 특이)
```

### 주요 메서드

```python
fit_from_dataframe(df)          # 학습
detect_from_dataframe(df)       # 전체 데이터 탐지 결과 반환
top_anomalies(df, n=30)         # 전국 기준 특이 거래 TOP N
summarize_anomalies(df)         # 전체 요약 통계
save(path) / load(path)
```

### 사용 예시

```python
import pandas as pd
from models.anomaly.anomaly_transaction_model import AnomalyTransactionModel

df = pd.read_csv("data/raw/Apart Deal_6.csv", encoding="utf-8", low_memory=False)
df["거래금액"] = df["거래금액"].astype(str).str.replace(",", "", regex=False).astype(float)

model = AnomalyTransactionModel(contamination=0.03, sample_size=200_000, random_state=42)
model.fit_from_dataframe(df)

top10 = model.top_anomalies(df, n=10)
print(top10[["시군구", "아파트", "거래금액", "평당가", "anomaly_score", "anomaly_rank"]])
```

## 3. seoul_anomaly_transaction_model

### 파일 위치

```text
models/anomaly/seoul_anomaly_transaction_model.py
```

### 클래스명

```python
SeoulDistrictAnomalyModel
```

### 목적

서울 데이터만 추출하여 **25개 구별로 IsolationForest를 독립 학습**합니다.  
"강남구 기준 특이 거래", "구로구 기준 특이 거래"처럼 구마다 독립된 기준으로 이상치를 판단합니다.

### 서울 데이터 필터 조건

```text
지역코드가 "11"로 시작하거나 시군구에 "서울"이 포함된 행
```

### 주요 입력 피처

```text
거래금액      전용면적      평당가        층
건물연식      기준금리      위도          경도
인근학교수    인근역수      세대수        거래연도
거래월
(구별 모델이므로 지역코드/시군구는 상수 → 피처 제외)
```

### 주요 결과 컬럼

```text
구명              시군구에서 추출한 구 이름
anomaly_raw_label 1=정상, -1=특이
anomaly_label     "정상거래" / "특이거래"
anomaly_score     값이 작을수록 더 특이 (구 내 상대 점수)
anomaly_rank      구 내 특이 순위 (1 = 해당 구에서 가장 특이)
district_rank     서울 전체 기준 순위
```

### 주요 메서드

```python
fit_from_dataframe(df)              # 구별 학습
detect_from_dataframe(df)           # 탐지 결과 반환
top_anomalies_by_district(df, n=10) # 구별 TOP N씩 반환 (25구 × N행)
top_anomalies(df, n=50)             # 서울 전체 기준 TOP N
summarize_by_district(df)           # 구별 요약 테이블
save(path) / load(path)
```

### 사용 예시

```python
import pandas as pd
from models.anomaly.seoul_anomaly_transaction_model import SeoulDistrictAnomalyModel

df = pd.read_csv("data/raw/Apart Deal_6.csv", encoding="utf-8", low_memory=False)
df["거래금액"] = df["거래금액"].astype(str).str.replace(",", "", regex=False).astype(float)

model = SeoulDistrictAnomalyModel(contamination=0.03, random_state=42)
model.fit_from_dataframe(df)

summary = model.summarize_by_district(df)
print(summary)

top = model.top_anomalies_by_district(df, n=10)
print(top[["구명", "아파트", "거래금액", "평당가", "anomaly_score", "anomaly_rank"]])
```

## 4. location_anomaly_transaction_model

### 파일 위치

```text
models/anomaly/location_anomaly_transaction_model.py
```

### 클래스명

```python
LocationAnomalyModel
```

### 목적

전국 **246개 시군구별로 IsolationForest를 독립 학습**합니다.  
`seoul_anomaly`의 전국 확장판으로, 각 시군구 내부 패턴 기준의 이상치를 탐지합니다.  
시군구별 TOP 1을 먼저 추출한 뒤 전체 TOP 10을 반환하는 방식으로 운용합니다.

### 학습 제외 조건

```text
시군구별 거래 건수가 30건 미만인 경우 학습에서 제외
실행 결과 기준: 경상북도 울릉군, 전라남도 신안군 제외 → 244개 시군구 학습
```

### 주요 입력 피처

```text
거래금액      전용면적      평당가        층
건물연식      기준금리      위도          경도
인근학교수    인근역수      세대수        거래연도
거래월
(시군구별 모델이므로 지역코드 피처 제외)
```

### 주요 결과 컬럼

```text
anomaly_raw_label   1=정상, -1=특이
anomaly_label       "정상거래" / "특이거래" / "미탐지(학습데이터부족)"
anomaly_score       값이 작을수록 더 특이 (시군구 내 상대 점수)
anomaly_rank        시군구 내 특이 순위 (1 = 해당 시군구에서 가장 특이)
total_rank          전국 기준 순위
```

### 주요 메서드

```python
fit_from_dataframe(df)                  # 시군구별 학습
detect_from_dataframe(df)               # 탐지 결과 반환
top_anomalies_top1_per_region(df, n=10) # 시군구별 TOP1 → 전체 TOP N
summarize_by_location(df)               # 시군구별 요약 테이블
save(path) / load(path)
```

| 종로구 | 경희궁자이(3단지) | 187,000 | 7,274 | -0.103268 | 1 |
| 중구 | 상지리츠빌장충동카일룸 | 590,000 | 7,955 | -0.117362 | 1 |

**해석:**  
모든 구에서 contamination=0.03 기준 약 3%씩 특이거래가 탐지됩니다.  
특이거래는 구별로 정상거래 평균 대비 평당가가 약 1,000~4,600만원 높습니다.  
성동구와 서초구는 정상거래 대비 특이거래 평당가 차이가 3,000~4,600만원으로 가장 큽니다.  
구로구 1위(한양수자인에듀힐즈, score=-0.140905)는 `location` 모델 전국 TOP 4와 동일 거래로, 구로구 내부 기준으로도 강하게 이상치로 포착됩니다.  
결과가 25구 × 10건 = 250행으로 데이터 규모가 크므로 Streamlit에서는 구 선택 필터 UI 적용을 권장합니다.

## 6. anomaly_score 해석 기준

| 범위 | 의미 |
|---|---|
| 음수, 절댓값 클수록 | 더 특이한 거래. 이상치에 가까움 |
| -0.15 이하 | 강한 이상 신호 (`location` 모델 기준 TOP권) |
| -0.10 ~ -0.15 | 중간 이상 신호 |
| -0.03 이하 | 약한 이상 신호 (`anomaly` 전국 단일 모델 기준 TOP권) |
| 0 근처 | 경계선 거래 |
| 양수 | 정상 패턴 거래 |
| `anomaly_raw_label = -1` | IsolationForest가 이상치로 분류 |
| `anomaly_rank = 1` | 해당 범위(전국/구/시군구) 내 가장 특이 |

> 모델마다 score 범위가 다르므로 서로 다른 모델의 score를 직접 비교하면 안 됩니다.

### 8.3 고가 이상치 vs 저가 이상치

이상치는 고가만 탐지하지 않습니다. 해당 지역 평균과 크게 다른 방향이면 저가도 이상치입니다.

```text
고가 이상치 예시
  - 아크로서울포레스트 130억 (성동구 내 극단 고가)
  - PH129 145억 (강남구 내 극단 고가)

저가 이상치 예시
  - 강북구 두온리치벨리 6억, 평당가 1,380 (서울 평균 대비 극단 저가)
  - 강북구 성원상떼빌 4.7억, 평당가 1,825
```

저가 이상치는 급매·특수 거래·소형 노후 단지 등에서 주로 나타납니다.  
발표 시 "특이 거래 = 고가 거래"라는 오해를 방지하고, 양방향 이상치임을 명시하는 것이 좋습니다.
