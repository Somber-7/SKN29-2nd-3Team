# 모델 정리

아파트 실거래가 데이터 프로젝트에서 지금까지 작성한 모델 파일 정리입니다.  
전체 구조는 `models/base.py`의 `BaseModel`, `BaseClusterModel`을 상속하는 방식으로 통일하는 것을 기준으로 합니다.

---

## 1. 공통 베이스 클래스

### 파일 위치

```text
models/base.py
```

### 역할

모든 모델이 동일한 인터페이스를 가지도록 하는 공통 추상 클래스입니다.

### 주요 클래스

```python
BaseModel
BaseClusterModel
```

### BaseModel 필수 메서드

```python
fit(X_train, y_train, **kwargs)
predict(X)
evaluate(X_test, y_test)
```

### BaseClusterModel 필수 메서드

```python
fit(X, **kwargs)
predict(X)
evaluate(X)
```

### 사용 목적

- 회귀 모델 공통 인터페이스 통일
- 분류 모델 공통 인터페이스 통일
- 군집화 모델 공통 인터페이스 통일
- Streamlit 화면에서 모델을 일관되게 호출하기 위함

## 2. 기본 아파트 거래금액 예측 모델

### 파일 위치

```text
models/regression/apartment_price_model_base.py
```

### 클래스명

```python
ApartmentPriceModel
```

### 모델 목적

아파트의 면적, 층, 건물연식, 지역, 금리, 주변 인프라 정보를 바탕으로 `거래금액`을 예측하는 회귀 모델입니다.

### 상속 구조

```python
from models.base import BaseModel

class ApartmentPriceModel(BaseModel):
    ...
```

### 사용 모델

```python
HistGradientBoostingRegressor
```

### 주요 입력 피처

```text
전용면적
층
건물연식
지역코드
기준금리
인근학교수
인근역수
세대수
브랜드여부
```

### 내부 생성 피처

```text
건물연식 = current_year - 건축년도
```

### 주요 메서드

```python
fit(X_train, y_train)
predict(X)
evaluate(X_test, y_test)
fit_from_dataframe(df)
evaluate_dataframe(df)
predict_series(df)
permutation_importance(df)
save(path)
load(path)
```

### 사용 예시

```python
import pandas as pd
from models.regression.apartment_price_model_base import ApartmentPriceModel


df = pd.read_csv("Apart Deal_6.csv")

model = ApartmentPriceModel(sample_size=100_000, random_state=42)
metrics = model.fit_from_dataframe(df)

print(metrics)

importance = model.permutation_importance(df, n_samples=10_000)
print(importance)

model.save("apartment_price_model.joblib")
```

### 특징

- 기존 `eda.ipynb` 모델링 로직을 클래스화
- `BaseModel` 상속 구조 적용
- 원본 DataFrame을 바로 넣는 편의 메서드 제공
- 원-핫 인코딩된 `지역코드`를 다시 합칠 필요 없이 원본 피처 기준 permutation importance 계산 가능

```text

거래월 = 거래일에서 추출
```


### 모델 비교 함수

```python

```

```python
    RandomForestPriceModel(sample_size=100_000),

    XGBoostPriceModel(sample_size=100_000),

result_df = compare_regression_models(models, df)
```


### 목적

아파트의 가격, 입지, 단지 규모, 주변 인프라 특성을 바탕으로 브랜드 등급을 분류하는 모델입니다.

### 상속 구조

```python
from models.base import BaseModel
```

### 지원 모델 타입

```python
model_type="lightgbm"
model_type="random_forest"
```

### 주요 입력 피처

```text
거래금액
전용면적
층
건물연식

위도

```

### 제외한 컬럼

```text
브랜드
브랜드여부
아파트
```

### 제외 이유

`브랜드`, `브랜드여부`, `아파트`를 피처로 넣으면 정답 유출이 발생할 가능성이 큽니다.  
예를 들어 아파트명에 `래미안`, `자이`, `푸르지오` 같은 단어가 포함되면 모델이 입지나 가격 특성을 학습하는 것이 아니라 이름만 보고 정답을 맞힐 수 있습니다.

### 주요 메서드

```python
fit(X_train, y_train)
predict(X)
predict_proba(X)
evaluate(X_test, y_test)
fit_from_dataframe(df)
predict_dataframe(df)
classification_report_dataframe(df)
get_feature_importance_df()
save(path)
load(path)
```

### 모델 비교 함수

```python
compare_brand_grade_classifiers(df, model_types=("lightgbm", "random_forest"))
```

### 사용 예시

```python
import pandas as pd
from models.classification.brand_grade_classifier import BrandGradeClassifier


df = pd.read_csv("Apart Deal_6.csv")

model = BrandGradeClassifier(
    model_type="lightgbm",
    sample_size=100_000,
    random_state=42,
)

metrics = model.fit_from_dataframe(df)
print(metrics)

report_df = model.classification_report_dataframe(
    df.sample(10_000, random_state=42)
)
print(report_df)

importance_df = model.get_feature_importance_df(top_n=20)
print(importance_df)

model.save("brand_grade_classifier.joblib")
```

## 9. 발표용 핵심 메시지

이번 프로젝트의 모델 구성은 다음과 같이 설명할 수 있습니다.

```text
본 프로젝트는 아파트 실거래가 데이터를 기반으로
거래금액 예측, 브랜드 등급 분류, 입지 유형 군집화를 수행했습니다.

회귀 모델에서는 Linear Regression, RandomForest, LightGBM, XGBoost를 비교하여
대용량 tabular 데이터에서 어떤 모델이 가장 안정적인 성능을 보이는지 확인했습니다.

분류 모델에서는 브랜드명을 직접 입력하지 않고,
가격, 입지, 주변 인프라, 단지 규모만으로 브랜드 등급을 예측하도록 설계했습니다.

군집화 모델에서는 위도, 경도, 평당가, 역/학교 접근성, 세대수 등을 활용하여
아파트를 유사한 입지 유형으로 묶고 지도 기반으로 시각화할 수 있도록 구성했습니다.
```
