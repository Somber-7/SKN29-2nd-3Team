## 1. 브랜드 등급 분류 모델 (BrandGradeClassifier)

### 목적

브랜드명을 직접 입력하지 않고, **입지·가격·단지 규모·주변 인프라 특성만으로** 아파트의 브랜드 등급을 예측하는 다중 분류 모델입니다.

### 브랜드 등급 정의

| 등급 | 해당 브랜드 예시 |
|---|---|
| 프리미엄 | 래미안, 자이, 힐스테이트, 아이파크, 푸르지오, 더샵, 롯데캐슬, 아크로 등 |
| 일반브랜드 | 위브, 하늘채, 호반, 우미, 효성, 중흥, 반도 등 |
| 공공(LH) | LH, 주공 |
| 기타 | 위 3개 외 모든 단지 |

### 알고리즘

- **XGBoost** (GPU 학습, CUDA 없으면 CPU 자동 폴백)
- 500만 건 전체 데이터 학습

### 주요 피처

```
거래금액, 전용면적, 층, 건물연식, 기준금리
위도, 경도, 인근학교수, 인근역수, 세대수
거래연도, 거래월, 지역코드, 시군구
```

> 브랜드, 브랜드여부, 아파트명은 **정답 누출 방지를 위해 피처에서 제외**

### 타깃 누출 방지 설계

브랜드명(아파트 컬럼)이나 브랜드여부를 피처로 넣으면 이름만 보고 정답을 맞히는 문제가 발생합니다.  
이를 방지하기 위해 순수하게 **가격·입지·단지 특성만으로 브랜드 등급을 예측**하도록 설계했습니다.

### 사용 방법

```python
from utils.db import load_apart_deals
from models.classification.brand_grade_classifier import BrandGradeClassifier

df = load_apart_deals()
model = BrandGradeClassifier()
metrics = model.fit_from_dataframe(df)
print(metrics)

# 예측
labels = model.predict_dataframe(df)
```

### 발표 설명

> 브랜드명을 직접 맞히는 모델이 아니라, 입지·가격·단지 규모·주변 인프라 특성을 바탕으로  
> 해당 아파트가 어떤 브랜드 등급에 가까운지 분류하는 모델입니다.

## 2. 지역 군집화 모델 (TorchKMeansLocationClusterModel)

### 목적

전국 아파트 실거래 데이터를 **위치·가격·거래 활성도 특성 기준으로 군집화**하여,  
비슷한 입지 특성을 가진 거래를 하나의 군집으로 묶습니다.

### 최종 설정

| 항목 | 값 |
|---|---|
| 알고리즘 | PyTorch GPU 전체 배치 KMeans |
| k (군집 수) | **7** |
| 피처 | 위도, 경도, 평당가, 건물연식, 거래활성도 |
| 위도/경도 가중치 | **×5** |
| n_init | **10** |
| max_iter | 300 |
| **Silhouette** | **0.4694** |
| **Davies-Bouldin** | **0.8479** |
| **Calinski-Harabasz** | **61,227** |

### 피처 설명

| 피처 | 계산식 | 역할 |
|---|---|---|
| 위도, 경도 | 원본 | 지리적 위치 (가중치 ×5 적용) |
| 평당가 | log1p(거래금액 / (전용면적 / 3.3)) | 가격 수준 |
| 건물연식 | 2026 - 건축년도 | 노후도 |
| 거래활성도 | 아파트별 거래건수 / 전체 거래건수 | 단지 인기도·유동성 |

### 거래활성도 데이터 누수 방지

- **fit 시**: 아파트별 거래비율을 `apt_activity_map_`으로 저장
- **predict 시**: 저장된 맵으로 매핑, 미등록 단지는 중앙값으로 대체

### MiniBatch → GPU 전체 배치 전환 이유

| 항목 | MiniBatchKMeans | TorchKMeans GPU |
|---|---|---|
| 수렴 방식 | 미니배치 (불완전 수렴) | 전체 배치 (완전 수렴) |
| 최종 Silhouette | 0.3617 | **0.4694** |
| 학습 시간 | 64.7초 | 45초 |

미니배치는 구조적으로 불완전 수렴하기 때문에, GPU 메모리(8GB)에 전체 데이터(500만×5피처 ≈ 100MB)를 올려 완전 수렴을 보장하는 방식으로 전환했습니다.

### 최종 군집 해석

| cluster | 거래건수 | 비율 | 평균평당가 | 권역 해석 |
|---|---|---|---|---|
| 2 | 2,330,822 | 47% | 1,814만 | 서울·수도권 |
| 5 | 778,648 | 16% | 1,032만 | 부산 외곽 주거지 |
| 3 | 755,952 | 15% | 847만 | 충청·전북권 |
| 4 | 371,492 | 8% | 952만 | 창원·김해 역세권 대단지 |
| 6 | 267,446 | 5% | 864만 | 전남·광주권 |
| 0 | 238,234 | 5% | 668만 | 강원·경북권 |
| 1 | 183,209 | 4% | 705만 | 전남 동부 (순천·여수·광양, 역세권 없음) |

> cluster 1 (순천·여수·광양): 전남에서 거래가 활발하고 인구도 많지만 지하철이 없는 지역 — 거래활성도는 높고 인근역수=0 이라는 독립된 특성으로 별도 분리됨

### 사용 방법

```python
from utils.db import load_apart_deals
from models.clustering.torch_kmeans_models import TorchKMeansLocationClusterModel

df = load_apart_deals()
model = TorchKMeansLocationClusterModel(n_clusters=7)
model.fit_from_dataframe(df)

print(model.metrics_)
summary = model.summarize_clusters(df)
print(summary)

# 새 데이터 예측
labels = model.predict(new_df)
```

### 발표 설명

> 전국 500만 건 실거래 데이터를 위치·가격·거래 활성도 특성으로 7개 권역으로 군집화했습니다.  
> PyTorch GPU 전체 배치 KMeans를 활용해 완전 수렴을 보장했으며,  
> sklearn MiniBatchKMeans 대비 Silhouette 기준 2.65배 개선(0.177 → 0.4694)을 달성했습니다.

## 3. 성능 개선 여정 요약

### 분류 모델

| 가중치 최적화 | 위도/경도 ×5, n_init=10 | 0.4694 |
| **최종** | **k=7 확정** | **0.4694** |

## 4. 군집 레이블 활용 방안

군집화 모델의 결과(cluster 레이블)는 단독으로 끝나는 것이 아니라 다른 모델의 입력으로 연결될 수 있습니다.

### 4.1 거래가격 예측 모델 피처로 활용
- 회귀 모델(LightGBM, XGBoost)에 `cluster` 레이블을 피처로 추가
- 위도/경도를 직접 넣는 것보다 군집 레이블이 과적합 위험 낮음
- 지역 프리미엄 정보를 압축된 형태로 반영 가능

### 4.2 군집별 독립 예측 모델
- 각 군집마다 별도 회귀 모델 학습
- 서울·수도권 군집은 고가 아파트 패턴에 특화
- 지방 소도시 군집은 해당 지역 패턴에 특화

### 4.3 Streamlit 지도 시각화
- 군집별 색상 구분 지도
- 군집별 평균 평당가·거래금액 비교
- 군집별 특성 요약 테이블
