## 2. 공통 컴포넌트 (`app/utils/ui.py`)

모든 페이지에서 아래 함수들을 import해서 사용합니다.

| 함수 | 용도 |
|------|------|
| `load_css()` | style.css 로드 — 각 페이지 최상단 필수 |
| `render_sidebar()` | 로고 + 전체 네비게이션 메뉴 — 각 페이지 최상단 필수 |
| `page_header(title)` | 페이지 제목 헤더 + 구분선 |
| `section_badge(icon, title, color)` | 🔵 섹션명 ─── 스타일 구분 배지 |
| `stat_card(value, label, sub)` | 통계 숫자 카드 HTML 반환 |
| `info_card(label, value, sub)` | 상관계수/정보 카드 HTML 반환 |
| `chart_card_open()` / `chart_card_close()` | 차트 래퍼 박스 (쌍으로 사용) |

### 새 페이지 추가 시 기본 템플릿

```python
import streamlit as st, sys, os
sys.path.insert(0, os.path.join(os.path.dirname(__file__), ".."))
from utils.ui import load_css, render_sidebar, page_header, section_badge

st.set_page_config(page_title="페이지명", layout="wide")
load_css()
render_sidebar()
page_header("페이지 제목")
```

> **사이드바에 새 페이지 추가 시** → `app/utils/ui.py`의 `render_sidebar()` 안에  
> `st.page_link("pages/파일명.py", label="아이콘  메뉴명")` 한 줄 추가

## 3. 페이지별 구현 현황

### 🏠 Home.py — 개요
- 통계 카드 4개 (총 거래건수 실데이터 · 나머지 3개 더미)
- 전체 거래 추이 차트 영역 (더미 placeholder)

### 📈 1_가격추이분석.py
- 시/도 → 구/군 2단계 selectbox 필터
- 집계 단위 선택 (월별/분기별/연별)
- 가격 추이 라인차트 (더미 랜덤 데이터)

### 📍 2_입지분석.py
- 상관계수 카드 5개: 기준금리 · 인근역수 · 인근학교수 · 세대수 · 건축연식
- 역세권/비역세권 · 학군지/비학군지 평균 평당가 차트 영역 (더미 placeholder)

### 🗺️ 3_지도시각화.py
- Plotly Choropleth **SVG 지도** (외부 타일 없음, 배경 투명)
- 17개 시/도 색상 코딩: 파랑(저가) → 초록(중간) → 빨강(고가)
- **3단계 드릴다운** selectbox: 시/도 → 시/군/구 → 읍/면/동
- Scattergeo 가격 원 마커 (크기·색상 = 가격 반영)
- 아파트명 검색창 (UI만 구현, 기능 미연결)
- 선택 지역 요약 카드 + 가격 범례
- GeoJSON: southkorea-maps 공개 데이터 (`@st.cache_data` 캐싱)

### 📊 4_회귀모델.py
- **모델 탭:** Linear · RandomForest · LightGBM · XGBoost
- 예측 조건 입력 폼 (시/도, 면적, 층, 건축연도, 브랜드 등급)
- 예측 거래금액 결과 카드 (더미 공식 계산)
- 성능 지표 카드: MAE · RMSE · R² (더미)
- 차트: 예측 vs 실제 산점도 · 피처 중요도 바 차트 (더미)

### 🏷️ 5_분류모델.py
- **모델 탭:** SVM · DecisionTree · Ensemble
- 하이퍼파라미터 설정 UI (class_weight, 커널, depth 등)
- 예측 브랜드 등급 + 확률 프로그레스 바 (더미)
- 성능 지표 카드: Accuracy · Precision · Recall · F1 (더미)
- 차트: 혼동 행렬 히트맵 · ROC-AUC 곡선 (더미)

### 🔵 6_군집화.py
- **모델 탭:** KMeans · DBSCAN · Agglomerative
- 파라미터 설정 UI (k, eps, min_samples, linkage 등)
- PCA 2D 군집 산점도 (**sklearn 직접 연산**, 더미 데이터 사용)
- 성능 지표: Silhouette Score · Davies-Bouldin (더미 데이터 기준 실제 계산)
- 군집별 평균 거래금액 바 차트

### 🧠 7_신경망.py
- **목적 탭:** 회귀(가격예측) · 분류(브랜드등급)
- 네트워크 구조 설정 UI (은닉층, 뉴런, Dropout, LR, BatchNorm, EarlyStopping)
- 네트워크 구조 텍스트 미리보기
- 학습 곡선 시각화 Train/Val Loss + Early Stop 수직선 (더미)
- 최종 성능 지표 · 예측 결과 시각화 (더미)

### 📋 8_모델비교.py
- **탭:** 회귀 모델 비교 · 분류 모델 비교
- 성능 비교표 (최고 성능 행 파란 배경 강조, 더미)
- 회귀: R² 가로 바 차트 · MAE/RMSE 그룹 바 차트
- 분류: F1 가로 바 차트 · 레이더 차트

## 5. 백엔드 연결 체크리스트

각 파일에서 `# ── 백엔드 연결 포인트` 주석 블록을 찾아 순서대로 교체합니다.

### Step A — DB 데이터 로드 (전 페이지 공통)

```python
# utils/db.py 이미 구현됨
from utils.db import fetch_all

rows = fetch_all("""
    SELECT 시도, 구군, 전용면적, 층, 건축연도, 거래금액, 브랜드
    FROM apart_deal WHERE 시도 = %s
""", (sido,))
df = pd.DataFrame(rows)
```

### Step B — 전처리 (4_회귀모델, 5_분류모델 공통)

```python
# utils/preprocessor.py 이미 구현됨
from utils.preprocessor import parse_price, parse_date, fix_floor, map_brand_grade, impute

df = parse_price(df)
df = parse_date(df)          # → 거래년도, 거래월, 거래분기 파생 컬럼 추가
df = fix_floor(df)           # → 층 음수값 NaN 처리
df = map_brand_grade(df)     # → brand_grade 컬럼 추가 (프리미엄/일반/공공/기타)
df = impute(df, strategy="median")
```

### Step C — 모델 로드 및 예측

| 페이지 | 저장 경로 (예시) | 로드 방법 |
|--------|-----------------|-----------|
| 4_회귀모델 | `models/regression/{model}.pkl` | `joblib.load(path)` |
| 5_분류모델 | `models/classification/{model}.pkl` | `joblib.load(path)` |
| 6_군집화 | `models/clustering/{model}.pkl` | `joblib.load(path)` |
| 7_신경망 | `models/neural_network/dnn_{task}.pt` | `torch.load(path)` |

```python
from joblib import load
model  = load(f"models/regression/{model_key}.pkl")
y_pred = model.predict(X_test)
```

### Step D — 성능 지표 교체


## 6. 연결 권장 순서

```
1. conf/.env 작성 → utils/db.py 연결 테스트
2. utils/preprocessor.py 전처리 파이프라인 검증
3. 모델 학습 완료 → .pkl / .pt 파일 저장
4. 4_회귀모델.py 백엔드 연결 (구조 가장 단순)
5. 5_분류모델.py 백엔드 연결
6. 8_모델비교.py 실제 지표로 교체
7. 1_가격추이분석.py 실데이터 연결
8. 2_입지분석.py 상관계수 실계산 연결
9. 3_지도시각화.py 아파트 검색 기능 구현
10. Home.py 통계 카드 실데이터 연결
```
